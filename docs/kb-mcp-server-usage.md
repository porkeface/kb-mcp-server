# kb-mcp-server 使用指南

> 本文档供 YI-AI 项目的 Agent 参考，了解如何使用易学知识库。

---

## 一、kb-mcp-server 是什么

kb-mcp-server 是一个 **MCP 知识库服务器**，为 Agent 提供易学经典文献的语义检索能力。

**核心定位**：YI-AI 项目的**知识操作系统层**（Single Source of Truth）。

**不是**：普通 RAG 问答系统
**而是**：为 Agent 写代码时提供权威易学知识来源

---

## 二、为什么要用它

Agent 在实现易学功能时经常出现：
- 六亲错误
- 纳甲错误
- 世应错误
- 旺衰判断错误
- 规则理解不一致

**原因**：Agent 没有统一的易学知识来源，会混合网络资料、开源项目、训练语料、自身推测。

**解决方案**：Agent 写代码前先查询 kb-mcp-server，获取权威规则。

## 四、可用 MCP Tools

Agent 可以调用以下工具：

### 4.1 kb_search — 语义搜索

```
kb_search(kb="yijing", query="纳甲规则", top_k=5)
```

搜索知识库中与查询最相关的内容。

**参数**：
- `kb`：知识库名称，固定为 `"yijing"`
- `query`：搜索关键词（自然语言）
- `top_k`：返回结果数量（默认 5）

**返回**：匹配的文本片段、相似度分数、来源书籍

---

### 4.2 kb_list — 列出知识库

```
kb_list()
```

列出所有可用的知识库。

---

### 4.3 kb_info — 知识库详情

```
kb_info(kb="yijing")
```

获取知识库的详细信息（文档数、分块数等）。

## 五、使用流程

**正确做法**：
```
1. Agent 收到任务（如"实现纳甲算法"）
2. 调用 kb_search(kb="yijing", query="纳甲规则")
3. 获取权威规则
4. 根据规则实现代码
5. 用 kb_search 验证实现是否正确
```

**错误做法**：
```
1. Agent 收到任务
2. 根据模型记忆直接写代码  ← 错！
3. 实现结果不稳定
```

## 七、注意事项

1. **先查后写** — 实现任何易学功能前，先搜索获取权威规则
2. **来源优先** — 优先参考《卜筮正宗》《增删卜易》《火珠林》的内容
3. **不要推断** — 不要根据模型记忆推断六爻规则，必须查询知识库
4. **案例验证** — 实现完成后，用《增删卜易》的案例验证正确性

## 九、长期架构

```
YI-AI
├── kb-mcp-server      ← 知识层（你在这里）
│      Knowledge Layer
│      "规则是什么"
│
├── Yi Rule Engine      ← 规则层
│      Rule Layer
│      "规则如何执行"
│
├── Agent               ← 推理层
│      Semantic Layer
│      "解释结果"
│
└── UI                  ← 产品层
       Product Layer
```

知识库负责告诉 Agent "规则是什么"，规则引擎负责"规则如何执行"，Agent 负责"解释结果"。
